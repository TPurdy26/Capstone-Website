CSS Basics

  • CSS styles HTML
  • Each rule is made up of a selector and a semicolon-separated list of declarations, with each of those declarations being made up of a property-value pair
  • Selectors (they refer to the HTML elements to which CSS rules apply):
  • Universal selector - *
  • Type selectors - [name of element]
  • Class selectors - .[name of class] (Can add multiple classes to an element by separating them with commas)
  • ID selectors - #[name of id]
  • Group separators - selectors can be separated between commas
  • Chaining selectors - .[chain].[specific chain] (can have things with same selectors but different subselectors)(can also do “.selector#chain” for ID selectors)
  • Descendant Combinator - uses the family tree from HTML, ancestor and child separated by a space (.ancestor .child)
  • Some properties have a default value set by the browser
  • color - changes an element’s text color
  • background-color - sets background color
  • font-family - declares font (can put a list for backups)
  • font-size - sets the size of the font
  • font-weight - affects boldness of the text
  • text-align - aligns text horizontally within an element, can use common keywords like “center”
  • height - sets height of an element
  • width - sets width of an element
  • External CSS - a separate file for the CSS and linking it inside of an HTML’s opening and closing "head" tags with a void "link>"element.
  • Example: "link rel=”stylesheet” href=”styles.css""
  • The href can either be an absolute url or a URL relative to the location of the HTMl file
  • Rank of precedence: ID , class, type (with type being at the bottom)
  • Things like * have no specificity, so everything takes precedence above it.
  • If there is no other tie-breaker, whichever came last takes precedence
  • Some elements can be “inherited” by child elements
